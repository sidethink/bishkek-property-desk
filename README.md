# Карта шоурумов

Небольшой MVP-прототип для риэлтора: интерактивная карта Бишкека, демо-объекты, фильтры и шоурум объекта с фото, видео/туром, факторами цены и похожими объектами.

## Что есть в первой версии

- Карта Бишкека на локально подключенном Leaflet.
- Премиальный слой Carto Voyager при доступном интернете.
- Локальная fallback-схема районов, если внешние тайлы недоступны.
- Демо-база объектов в `app.js`.
- Поиск по названию, району, типу дома и состоянию.
- Фильтры по району, цене, комнатности и наличию медиа.
- Переключатель "Все / Сделки / Объявления".
- Мини-метрики по отфильтрованным объектам.
- Шоурум выбранного объекта.
- Поддержка фото, YouTube iframe и 360/3D iframe.
- Простая логика похожих объектов и ценового диапазона.
- Расчет ориентировочной полной цены по похожим объектам.
- Режим "Презентация клиенту", который скрывает внутренние заметки.
- Мобильное открытие/закрытие шоурума без перекрытия карты.
- Скрытие левой витрины и режим чистой карты.
- Полноэкранный режим шоурума для планшета.
- Клиентская ссылка вида `?share=object-id`, которая показывает один объект.

## Как открыть

Откройте `index.html` в браузере.

Проект сделан статическим, без сборщика и backend, чтобы быстро проверить идею. На следующем этапе можно подключить Supabase, импорт CSV/Excel и реальные данные команды.

Leaflet CSS/JS лежат локально в `vendor/leaflet`, поэтому `index.html` можно открывать через `file://`. Для картографических тайлов все еще нужен интернет; если тайлы недоступны, включится локальная схема районов с кликабельными маркерами.

При открытии через `file://` кнопка "Ссылка" копирует только локальную демо-ссылку. Чтобы отправлять ссылку клиенту на другой телефон/планшет, проект нужно выложить на сайт или локальный сервер в одной сети.

## Где менять демо-данные

Все объекты лежат в массиве `properties` в `app.js`. Для реального объекта нужно заменить поля: координаты, цену, площадь, район, фото, YouTube-ссылку, ссылку на 360/3D-тур, факторы цены и клиентский вывод.
